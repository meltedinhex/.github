# Melted in Hex

**Open tooling and detections for malware analysis, reverse engineering, and threat hunting.**

Melted in Hex publishes practical, defensive security work — analysis workflows, runnable
tooling, and open detection content — alongside write-ups at
[meltedinhex.com](https://meltedinhex.com/).

## What's here

| Project | What it is |
|---|---|
| [**analyst-ai-pack**](https://github.com/meltedinhex/analyst-ai-pack) | An open agent-skills library — 118 curated, runnable skills for malware analysis, reverse engineering, and threat hunting, each mapped to MITRE ATT&CK, D3FEND, and CAR. |
| [**detections**](https://github.com/meltedinhex/detections) | Open detection coverage (YARA, Sigma, KQL) and IOCs for malware campaigns analyzed on the blog. |
| [**meltedinhex.github.io**](https://github.com/meltedinhex/meltedinhex.github.io) | Source for the [meltedinhex.com](https://meltedinhex.com/) blog. |

## Focus areas

- **Malware analysis** — static, dynamic, behavioral, and memory analysis of real-world samples.
- **Reverse engineering** — unpacking, deobfuscation, config extraction, and language-specific RE.
- **Threat hunting** — hypothesis-driven hunts across endpoint, network, and identity telemetry.
- **Detection engineering** — turning findings into durable YARA, Sigma, and KQL detections.

## Principles

- **Defensive by design.** Everything here supports detection, analysis, and response — not offense.
- **No live malware.** Repositories ship tooling and detections, never operational samples or payloads.
- **Open and reproducible.** Content is original, cited, and built to be run and verified.
